## 五、計算邏輯結構 ✅ Phase 5 已完成 Pipe Shoe 家族 pilot

### Phase 5 架構說明（給下一位 AI）

```
python_app/
├── configs/
│   └── pipe_shoe_spec.json          ← D-80 sizing rules + 6 component 定義
├── core/
│   ├── pipe_shoe_engine.py          ← 薄 engine（265 行），讀 spec 並 dispatch
│   └── types/
│       └── type_52.py               ← 3 行 wrapper，dispatcher only
```

**type_52.py 的 dispatch 邏輯**（types 52/53/54/55/66/67 全走同一個 engine）：
```python
def calculate(fullstring: str) -> AnalysisResult:
    type_id = get_part(fullstring, 1)   # "52", "53", ... "67"
    return pipe_shoe_engine.calculate(fullstring, type_id)
```

**pipe_shoe_spec.json 的 sizing 規則**：
- `A_by_size`：gusset plate 寬度（按管徑分段）
- `B_by_size`：gusset plate 厚度
- `C_by_size`：結構件規格（`"200*100*5.5"` / `"200*200*8"` / `"FB12"`）
- `D_by_size`：預設 LOPS 長度
- `E_by_size`：≥10" pad 半延伸量
- `pad_t_fallback_by_size`：pad 厚度規則（目前 ≤8" 用 Sch10S 管壁厚，待確認）

**component 列表**（6 個）：
| id | 名稱 | when 條件 |
|----|------|-----------|
| `pad` | Pad_52Type | `pad_symbol != N/A` |
| `wedge` | Angle 40×40×5 | `angle_wedge`（52/53/54/55=True，66/67=False）|
| `beam` | H Beam | `C != FB12`（≤14" 管）|
| `fab_bottom` | FB_52Type_1 | `C == FB12`（≥16" 管）|
| `fab_web` | FB_52Type_2 | `C == FB12`（≥16" 管）|
| `gusset` | FB_52Type_3 | `pipe_size >= 10` |

**手動驗證 5 案例**（Phase 5 完成後全部通過）：

| Input | Pad | 結構件 | Wedge | Gusset |
|-------|-----|--------|-------|--------|
| `52-1/2B-A-150-200` | 無 | H200×100×5.5 | ✓ | — |
| `52-2B(P)-A(A)-130-500` | 2.11mm（Sch10S） | H200×100×5.5 | ✓ | — |
| `66-10B(P)-A-150-250` | 9mm | H200×200×8 | — | 4× |
| `66-1.1/2B(P)-A-150-150` | 2.11mm（Sch10S） | H200×100×5.5 | — | — |
| `52-20B(P)-A-200-600` | 12mm | FAB T-section | ✓ | 4× |

## 九、重要的實作細節（給下一位 AI 的地雷清單）

1. **Edit 工具截斷大型檔案**  
   本 session 多次遇到 Edit 工具在修改大型檔案（validate_tables.py, pipe_shoe_engine.py）後截斷後半部。  
   **解法**：對 >500 行的檔案，優先用 bash `cat >` 或 Python script 整體重寫。

2. **pyc 快取無法刪除（Windows NTFS 掛載）**  
   `/sessions/.../mnt/` 下的 `.pyc` 以 `-rwx------` 模式掛載，sandbox 無法刪除。  
   **解法**：用 `py_compile.compile(...)` 強制重新編譯覆蓋；或 touch .py 讓 Python 認 stale。

3. **git index.lock 問題**  
   sandbox 內 git 操作有時留下 `index.lock`，需用戶在本機 PowerShell 執行 `Remove-Item .git\index.lock -Force`。

4. **float key 問題（bridge 模組）**  
   JSON 讀出的 dict key 是 `str`，但 `get_lookup_value()` 回傳 `float`，導致查詢 miss。  
   已修復的 9 個 bridge 檔用三路轉型 dict comprehension 解決。  
   若新增 bridge 模組，記得套用同樣模式。

5. **type_52 是 dispatcher，type_56 是獨立家族**  
   - `calculator.py` 中 52/53/54/55/66/67 全部指向 `type_52.calculate`
   - type_56 是 D-67 pipe stop，**不在** pipe shoe 家族，計算邏輯完全不同，**勿混淆**
